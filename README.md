# DevOps Defender

**Deploy unmodified Docker apps to hardware-verified Intel TDX enclaves. Zero stored secrets.**

[Website](https://devopsdefender.com) · [GitHub](https://github.com/devopsdefender/dd)

---

## The Problem

Your cloud provider can read your memory. Your CI secrets live in plaintext on someone else's machine. Your attack surface is every open port, every stored credential, every trusted intermediary.

DevOps Defender eliminates all three.

## Three Pillars

**Intel TDX** — The hardware proves itself. Every VM runs in encrypted memory that even the hypervisor cannot read. Cryptographic attestation quotes, generated by the CPU and verified by Intel, prove the machine is genuine before it touches your workload.

**GitHub OIDC** — Nothing to steal. Deployers authenticate with short-lived JWT tokens issued by GitHub Actions at deploy time. No API keys. No stored credentials. No secrets to rotate, leak, or exfiltrate.

**Cloudflare Tunnels** — Nothing to attack. Each agent gets an outbound-only encrypted tunnel. No open ports. No TLS certificates to manage. No firewall rules to misconfigure.

## How It Works

1. **Register** — Define your app, its Docker image, and the GitHub org authorized to deploy it.
2. **Publish** — Push your container image. DevOps Defender records the content digest for attestation.
3. **Attest** — The TDX hardware generates a cryptographic quote binding VM identity, firmware, kernel, and rootfs into an unforgeable measurement chain. Intel verifies it.
4. **Deploy** — Your container launches inside the verified enclave. The control plane monitors health and enforces attestation continuously.

If it runs in Docker, it runs in a TDX enclave. No SDK. No code changes. No special toolchains.

## Repository

```
dd/
├── agent/          # Rust daemon on TDX VMs — registration, attestation, workloads, heartbeats
├── control-plane/  # Rust/Axum API — orchestration, verification, health monitoring
├── images/         # Packer definitions for GCP agent VM images
├── infra/          # Ansible playbooks for GCP deployments
├── openapi/        # OpenAPI spec (drives agent code generation)
└── website/        # Landing page at devopsdefender.com
```

## Quick Start

```bash
# Control plane
DD_CP_DATABASE_URL=sqlite://dev.db DD_CP_ADMIN_PASSWORD=changeme cargo run --bin dd-cp

# Agent (skip attestation for non-TDX environments)
DD_AGENT_SKIP_ATTESTATION=true DD_CP_URL=http://localhost:8080 cargo run --bin dd-agent
```
