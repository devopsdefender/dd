Four kinds. No more.

A typed schema, four reference workloads. No “anything as JSON.”

₿

Oracle

Long-running attested service that signs and publishes data on-chain. Optional signed public log; anyone can verify the hash chain.

🧠

LLM

Local inference (Ollama, vLLM) or a TEE-protected proxy to upstream APIs. Upstream key never leaves the enclave. Optional client-encrypted history.

🖥

Shell

Per-user attested bash over the web. GitHub login at the control plane; the PTY runs inside the sealed VM. Optional encrypted recording.

🤖

Bot

Autonomous LLM loops with outbound integrations — Signal, WhatsApp, your own APIs — inside a TDX-sealed VM. Optional client-encrypted chat history.

How it works

1

Boot a sealed VM

Intel TDX measures memory. EasyEnclave is PID 1 — no systemd, no shell to attack, Unix-socket API.

2

Claim a vanity

Pick oracle, llm, shell, or bot and a DNS name. The control plane picks a host and points the CNAME.

3

Verify before you trust

Intel Trust Authority signs the TDX quote. Pull /health, check the quote, then send the prompt, key, or message.

Auth, in one line

Humans log in with GitHub at the control plane. CI deploys with GitHub Actions OIDC. Agents register with an Intel Trust Authority token. No stored secrets, anywhere.
