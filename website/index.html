Zero-Secret Trust Model

Three independent layers. No stored credentials. The hardware, the identity, and the network each prove themselves.

Intel TDX

Every VM runs in encrypted memory that even the hypervisor cannot read. Cryptographic attestation quotes, generated by the CPU and verified by Intel, prove the machine is genuine before it touches your workload.

GitHub OIDC

Deployers authenticate with short-lived JWT tokens issued by GitHub Actions at deploy time. No API keys. No stored credentials. No secrets to rotate, leak, or exfiltrate.

Cloudflare Tunnels

Each agent gets an outbound-only encrypted tunnel. No open ports. No TLS certificates to manage. No firewall rules to misconfigure.

How It Works

From code commit to verified enclave in four steps.

1

Register

Define your app, its Docker image, and the GitHub org authorized to deploy it.

2

Publish

Push your container image. DevOps Defender records the content digest for attestation.

3

Attest

The TDX hardware generates a cryptographic quote binding VM identity, firmware, kernel, and rootfs into an unforgeable measurement chain. Intel verifies it.

4

Deploy

Your container launches inside the verified enclave. The control plane monitors health and enforces attestation continuously.

Quick Start

Add the GitHub Action to your workflow. That's it.

.github/workflows/deploy.yml 
permissions:
  id-token: write

steps:
  - name: Deploy to TDX
    uses: devopsdefender/deploy-action@v1
    with:
      control-plane: https://app.devopsdefender.com
      app-name: my-application
      image: ghcr.io/myorg/myapp:${{ github.sha }}
